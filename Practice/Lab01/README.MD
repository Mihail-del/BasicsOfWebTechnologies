# Аудит доступності - broken.html
 
## Знайдені та виправлені проблеми
 
### 1. Відсутній `lang` на `<html>`
**Проблема:** `<html>` без атрибута `lang`.  
**Чому важливо:** Скринридери використовують `lang` для вибору мови синтезу мовлення.
**Виправлення:** `<html lang="uk">`
 
---
 
### 2. `<img>` без `alt`
**Проблема:** `<img class="avatar" src="me.jpg">` - немає атрибута `alt`.  
**Чому важливо:** Скринридер вголос читає ім'я файлу (`me.jpg`), що не несе змісту. **Виправлення:** `alt="Фото Олени"`
 
---
 
### 3. `<img>` без `alt`
**Проблема:** `<img src="calendar-screenshot.png">` - немає атрибута `alt`.  
**Чому важливо:** Користувачі скринридерів або з вимкненими зображеннями не знають, що зображено.  
**Виправлення:** `alt="Скриншот календаря"`
 
---
 
### 4. Заголовки в неправильному порядку
**Проблема:** Після `<h1>` одразу йде `<h3>` для розділу «Освіта».  
**Чому важливо:** Пропуск рівня порушує логічну структуру документа і ускладнює орієнтацію.  
**Виправлення:** `<h3>Освіта</h3>` замінено на `<h2>Освіта</h2>`
 
---
 
### 5. Список через `<br>` замість `<ul>` (освіта і технології)
**Проблема:** Перелік пунктів освіти та технологій оформлено переносами рядка `<br>`.  
**Чому важливо:** `<br>` - Скринридер не оголошує кількість елементів і не дозволяє навігувати по пунктах.  
**Виправлення:** Замінено на `<ul><li>...</li></ul>`
 
---
 
 
### 6. `<a>` без `href` і з `onclick`
**Проблема:** `<a onclick="alert('email')">Написати email</a>` - немає `href`.  
**Чому важливо:** Без `href` елемент не фокусується з клавіатури клавішею Tab, скринридер не оголошує його як посилання.
**Виправлення:** `<a href="mailto:olena@example.com">Написати email</a>`
 
---
 
### 7. `<div onclick>` як «кнопка»
**Проблема:** `<div class="btn" onclick="...">Зателефонувати</div>`  
**Чому важливо:** `<div>` не активується Enter/Space, скринридер не оголошує його як кнопку. 
**Виправлення:** Замінено на `<button type="button" class="btn">Зателефонувати</button>`
 
---
 
### 8. `<input>` без `<label>`
**Проблема:** `<input type="email" placeholder="Ваш email">` - немає пов'язаного `<label>`.  
**Чому важливо:** `placeholder` зникає під час введення тексту. Скринридер не може коректно оголосити призначення поля без `<label>`.
**Виправлення:** Додано `<label for="subscribe-email">Ваш email</label>` і `id="subscribe-email"` на `<input>`.
 
---
 
### 9. `<button>` без атрибута `type`
**Проблема:** `<button>Підписатися</button>` - без `type`.  
**Чому важливо:** За специфікацією HTML, `<button>` без `type` має неявний тип `submit`. Якщо кнопка опиниться всередині `<form>`, це призведе до небажаної відправки форми.  
**Виправлення:** `<button type="submit">Підписатися</button>`
